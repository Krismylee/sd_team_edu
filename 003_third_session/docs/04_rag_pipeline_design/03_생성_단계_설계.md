# 03. 생성 단계 설계

## 이 챕터에서 배우는 것

- 생성 단계의 역할과 책임
- 프롬프트 설계의 핵심 요소
- 출력 포맷과 안전장치

---

## 1. 생성 단계의 역할

생성 단계는 **검색 결과를 사용자 답변으로 변환**합니다.
문맥을 정확히 반영하고, 형식과 안전성을 지키는 것이 핵심입니다.

---

## 2. 프롬프트 설계 핵심 요소

- 질문 재진술로 **의도 명확화**
- 문맥 사용 규칙(출처 기반 답변)
- 답변 포맷(요약/목록/표 등)
- 금지된 행동(추측/허위 생성 금지)

---

## 3. 출력 포맷 설계

- **구조화된 출력**을 기본으로 설계
- JSON 또는 명확한 섹션 구분
- 에러/폴백 상황을 구분할 수 있어야 함

---

## 4. 안전장치

- 근거 없는 답변은 **거절 또는 폴백**
- 민감 정보 노출 방지
- 불확실성 표현 규칙 정의

---

## 5. 체크리스트

- 프롬프트가 의도를 명확히 전달하는가?
- 출력 포맷이 일관되게 유지되는가?
- 폴백 메시지까지 설계되었는가?
